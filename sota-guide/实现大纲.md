### 阶段一：数据准备与模式提取

**目标**：从收集的小红书帖子中提炼出“五阶段套路”和关键词库

**步骤**：

1. 收集20-30篇小红书“如何入门NLP/文科转码”帖子，存为txt文件（手动复制即可）
2. 写一个脚本对所有帖子进行分词、词频统计
3. 用共现分析或LDA找出五阶段的典型关键词
4. 手工归纳成五个阶段的词库：基础、ML课、NLP课、项目、目标

**向AI描述**：

> “我有20篇小红书入门NLP的帖子文本，存在`data/posts/`文件夹里。帮我写一个Python脚本，用jieba分词，统计所有帖子的词频，并生成词云。然后再做一个共现分析，看看哪些词经常一起出现，帮我找出五个主题簇。”

**产出文件**：

- `data/keywords_base.json`（数学/编程词）
- `data/keywords_ml.json`（ML课词）
- `data/keywords_nlp.json`（NLP课词）
- `data/keywords_project.json`（项目词）
- `data/keywords_goal.json`（目标词）

### 阶段三：生成逻辑实现（核心）

**目标**：实现根据用户选择生成骨架句的函数

**步骤**：

1. 写函数`generate_skeleton(goal_type, major=None)`：
   - 随机选择一个模板。
   - 从通用词库（math/ml/nlp）随机抽词填充。
   - 根据`goal_type`从`project[goal_type]`和`goal[goal_type]`随机抽词填充`{project}`和``{goal}``。
   - 如果`goal_type == 'major'`，将用户填写的专业名自然融入句子开头（方案C：直接说“XX领域现在有很多NLP应用…”）。
2. 返回完整的骨架句字符串。

**向AI描述**：

> “写一个Python函数`generate_skeleton(goal_type, major=None)`，它从`templates.json`随机选一个模板，从`vocab.json`里根据goal_type抽取对应词库填充math/ml/nlp/project/goal。如果是major类型，要把专业名自然地融入开头（例如‘医学领域现在有很多NLP的应用场景…’）。返回填充后的骨架句。”

### 阶段六：后端Flask整合

**目标**：连接前端、生成逻辑、模型润色，实现完整流程。

**路由设计**：

- `GET /` → 渲染首页
- `GET /quiz` → 渲染问卷页
- `POST /result` → 接收表单数据，生成结果，渲染结果页

**处理流程**：

1. 在`/result`中获取表单数据：`goal_type`, `major`（如果有）, `time`, `style`（后两个可选，可用于微调语气，但核心仍用词库）
2. 若`goal_type == 'curiosity'`，调用`generate_curiosity()`生成彩蛋文本。
3. 否则：
   - 调用`generate_skeleton(goal_type, major)`获得骨架句。
   - 调用`polish_with_icl(skeleton)`获得润色后的完整段落。
4. 渲染`result.html`，将生成的文本传入模板。

**彩蛋生成器`generate_curiosity()`**：

- 可用独立的模板+ICL（或直接用几个手写示例让模型模仿生成），主题围绕“问题驱动”：先有想法，再用技术，用什么学什么，不要怕没基础。
- 示例：“先想一个你感兴趣的问题，比如‘我想分析某部小说的人物关系’，然后看看NLP能不能帮上忙。用到什么学什么，不用提前学完所有理论。代码不会就问AI，慢慢就能做出来。”
- 返回生成的文本。

**向AI描述**：

> “在Flask应用中添加三个路由：
>
> - `/` 渲染首页
> - `/quiz` 渲染问卷页
> - `/result` 处理POST请求，参数包括goal_type等。
>   如果goal_type是‘curiosity’，调用`generate_curiosity()`生成彩蛋；否则调用`generate_skeleton`和`polish_with_icl`生成标准指南。最后渲染结果页，传入生成的文本。
>   注意模型在启动时加载一次，彩蛋生成器可以用ICL实现，避免加载第二个模型。”

**2. 模板设计（灵活组合）**

为了让项目和目标的关系既自然又多样，可以设计几种不同的模板句式：

- **强绑定型**：`动手做一个{project}，最后{goal}。`
- **因果型**：`通过{project}积累经验，这样就能{goal}。`
- **并列型**：`一边做{project}，一边朝着{goal}努力。`
- **项目隐含目标型**：`先尝试{project}，这个过程会帮你{goal}。`
- **目标前置型**：`如果想{goal}，可以从{project}开始。`

例如：

- 科研类：`如果能进组跟老师做研究，同时复现一些论文，后面发顶会就会顺很多。`
- 职业类：`做个完整的项目（比如电商推荐），然后拿去面试，拿到offer的概率就大了。`

**3. 润色阶段的灵活性**

模板生成的骨架句可能是：“动手做一个{project}，最后{goal}。” 然后ICL润色时会把它变成更自然的段落，比如：

- 职业版：“我当时就是做了一个情感分析项目，然后拿着它去面试，最后拿到了offer。”
- 科研版：“进组以后，我一边复现论文一边读顶会文章，后来真的发了一篇。”

这样，即使模板句式相同，润色后的文本也会因词库选择和ICL示例而呈现不同风格。

### 需要微调大纲的地方

在阶段三（生成逻辑）中，要确保`generate_skeleton`函数能正确处理`project`和`goal`的匹配。建议在函数中：

1. 根据`goal_type`从对应的`project`和`goal`子库中随机抽取。
2. 如果`goal_type`是“兴趣”或“不知道”，可以允许`goal`为空或使用开放式结尾（通过模板中省略`{goal}`或使用特殊占位符实现）。
3. 对于“专业结合”，除了融入专业名，`project`和`goal`仍从`major`子库抽取。

在阶段二（词库构建）中，需要将上述分层词库完整地写入`vocab.json`。



兴趣/未知类补充词库示例：

```json
"project": {
    "interest": [
        "手搓一个Transformer模型",
        "从头实现一个简单的NLP模型",
        "读几篇经典论文，搞懂原理",
        "跟踪最新的文献，看看前沿方向",
        "尝试改进一个小模型",
        "用wispaper搜索感兴趣的论文"
    ]
},
"goal": {
    "interest": [
        "真正理解模型为什么这么设计",
        "有能力看懂最新论文",
        "能提出自己的改进想法",
        "找到自己感兴趣的研究方向",
        "从‘会用’变成‘懂用’"
    ]
}
```
